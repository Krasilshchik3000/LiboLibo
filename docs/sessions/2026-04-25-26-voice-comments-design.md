# 2026-04-25-26 — Дизайн фазы 4: голос-комментарии к эпизодам

## Контекст

Фаза 2.3 (премиум через Adapty) задизайнена и частично выехала: бэкенд деплоится, iOS-сторона запланирована (см. сессию [`2026-04-25-25-ios-subscription-plan.md`](2026-04-25-25-ios-subscription-plan.md)). Илья пришёл с новой фичей: пользователь во время прослушивания эпизода жмёт микрофон на lock screen, наговаривает голос-комментарий с тайм-кодом, подкаст в этот момент плавно паузится, после паузы плавно возвращается, дальше комментарий автоматически расшифровывается через AI и публикуется в эпизоде, чтобы все могли и послушать, и прочитать.

## Что сделали

Чисто планировочная сессия, без кода. Через `superpowers:brainstorming` прошлись по архитектурным развилкам и записали полную дизайн-спеку фазы 4.

### Принятые решения

| Вопрос | Решение |
|---|---|
| Тип фичи | Публичные комментарии (видны всем слушателям эпизода). Запись — премиум-only, чтение — для всех. |
| Идентичность | `adapty_profile_id` (тот же, что в `entitlements`). Логина нет. Имя пользователя — случайная русская птица из pool ≈1000+ названий, детерминированная хэшем `profile_id`. На коллизию — суффикс `-2`. |
| Lock-screen UX | Live Activity с интерактивной кнопкой микрофона (App Intent, iOS 17+). |
| Жизненный цикл записи | Гибрид (single-shot + grace window). Тап → подкаст плавно пауза → юзер говорит → 1.5 с тишины → подкаст плавно резюм → 3 с soft cutoff (если снова заговорил — возвращаемся в запись одним непрерывным треком). |
| Транскрипция | `SFSpeechRecognizer`, `ru-RU`, on-device когда доступно, fallback на Apple-сервер. |
| Хранилище аудио | Railway Volume mount, `/data/comments/<uuid>.m4a`. |
| Лимиты | 60 сек длина, 2 МБ файл, 4000 символов transcript. |
| Модерация | На v1 — нет. Только владелец может удалить. |
| UI комментариев | Секция «Реплики слушателей» в `EpisodeDetailView` с тайм-кодами, маркеры на скрабере в `PlayerView`, мини-плеер. |

### Артефакт

**Спека:** [`docs/specs/step-04-voice-comments.md`](../specs/step-04-voice-comments.md) — полный дизайн: бэкенд-схема (User, Comment), эндпоинты, iOS-сервисы (`VoiceCommentRecorder`, `CommentsRepository`, `LiveActivityService`, `AudioSessionCoordinator`), state machine VAD, Live Activity и App Intent, UI, permissions, edge cases, безопасность, 30 шагов имплементации (бэк → iOS).

## Что НЕ делали

- Никакого кода. SPM, миграции, новый widget extension target, App Intent — всё в следующих сессиях.
- Не подключали Volume в Railway.
- Не составляли pool из 1000+ птиц — это шаг 2 имплементации.
- Декомпозицию по подфазам (3.0/3.1/3.2/3.3) рассмотрели, но Илья выбрал «делать сразу всё в одной спеке» — план имплементации внутри спеки разбит на 30 шагов, бэк сначала.
