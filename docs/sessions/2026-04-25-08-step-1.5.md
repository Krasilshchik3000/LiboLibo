## Что сделано

### 1. Поиск по гайдам iOS 18+

[`App/RootView.swift`](../../LiboLibo/App/RootView.swift) переведён со старого `View.tabItem(...)` на новый **Tab API**:

```swift
TabView(selection: $selectedTab) {
    Tab("Фид",      systemImage: "list.dash",            value: .feed)     { FeedView() }
    Tab("Подкасты", systemImage: "rectangle.grid.2x2",   value: .podcasts) { PodcastsView() }
    Tab("Моё",      systemImage: "person.crop.circle",   value: .profile)  { ProfileView() }
    Tab(value: .search, role: .search)                                     { SearchView() }
}
```

`Tab(role: .search)` — это и есть «последний гайд». На iOS 18+/26 он рендерится как **отдельная иконка лупы справа** от обычных табов, а на iOS 26 при тапе по нему таб-бар трансформируется в Apple-овский «коллапс»: список табов сжимается в значок ≡ слева, а справа разворачивается полноценное поле поиска. Ровно то, что Apple Music и Apple Podcasts.

`@TabContentBuilder<SelectedTab>` для общего блока табов, чтобы не дублировать четыре `Tab(...)` строки между modern и legacy путями.

### 2. Bug: пустое «облачко» плеера

В [`RootView`](../../LiboLibo/App/RootView.swift) `tabViewBottomAccessory { ... }` оставлял зарезервированную полоску даже когда `if let episode` внутри возвращал `EmptyView` — отсюда пустое «облачко» снизу. Фикс: накладываем модификатор только когда есть текущий эпизод.

```swift
if player.currentEpisode != nil {
    TabView { … }.tabViewBottomAccessory { MiniPlayerView()… }
} else {
    TabView { … }
}
```

`@State` `selectedTab` живёт ВНЕ TabView, поэтому переключение между «есть mini-player» и «нет mini-player» не сбрасывает выбранный таб.

### Калибровка координат

Окно симулятора в macOS — `(1077, 38, 415x890)` или после движения `(961, 52, 415x890)`. Маппинг скриншот→mac:

```
mac_x ≈ window_x + 9.3 + (screenshot_x / 1206) * 396.4
mac_y ≈ window_y + 28  + (screenshot_y / 2622) * 862
```

Empirical: для таб-бара mac_y ≈ window_y + 835; для первого ряда списка mac_y ≈ window_y + 250.

Если окно симулятора подвинется — координаты надо пересчитать через
`osascript -e 'tell application "System Events" to tell process "Simulator" to get position of window 1'`.
