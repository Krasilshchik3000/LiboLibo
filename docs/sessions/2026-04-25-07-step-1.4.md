# 2026-04-25 — Шаг 1.4: UX-фиксы (mini-player, сетка) и поиск (сессия 7)

**Контекст:** Илья прислал три замечания по двум скриншотам: (1) mini-player наезжает на таб-бар, нужен «нативный liquid glass и анимации Apple, как в системе, со скролл-сжатием»; (2) сетка подкастов «съехала», левая колонка стоит неровно; (3) добавить **объектный поиск**.

## Что сделано

### 1. Mini-player → нативный iOS 26 паттерн

[`App/RootView.swift`](../../LiboLibo/App/RootView.swift) разделён на два пути:

- **iOS 26+:**
  ```swift
  TabView { tabs }
      .tabBarMinimizeBehavior(.onScrollDown)
      .tabViewBottomAccessory {
          if player.currentEpisode != nil {
              MiniPlayerView()
                  .onTapGesture { showFullPlayer = true }
          }
      }
  ```
  Это Apple-blessed размещение: mini-player живёт в зоне над таб-баром, оба используют единый liquid-glass материал, при скролле вниз таб-бар автоматически сжимается. Никакого ручного `safeAreaInset`, никаких наездов.

- **iOS 18–25 fallback:** старый `safeAreaInset(edge: .bottom)` с `.regularMaterial` подложкой и `.transition(.move(edge: .bottom).combined(with: .opacity))` — для тех, у кого ещё не iOS 26.

[`MiniPlayerView`](../../LiboLibo/Features/Player/MiniPlayerView.swift) лишился собственного фона: на iOS 26 фон даёт сам `tabViewBottomAccessory`, на legacy его навешивает `RootView`. Вертикальные паддинги уменьшены — компактнее.

### 2. Сетка подкастов — выравнивание

[`PodcastsView`](../../LiboLibo/Features/Podcasts/PodcastsView.swift): причина «съезжающей» левой колонки — текстовые подписи разной высоты (1 vs 2 строки) меняли итоговую высоту тайлов и ломали ряды `LazyVGrid`. Фиксы:

- `Text(podcast.name).lineLimit(2, reservesSpace: true)` — резервируется место под 2 строки всегда, даже если имя короткое.
- `.frame(maxWidth: .infinity, alignment: .leading)` на тексте — однозначное выравнивание по левому краю.
- `LazyVGrid(alignment: .leading, ...)` и `GridItem(.adaptive(minimum: 110), spacing: 12, alignment: .top)` — тайлы прижимаются к топу строки, тексты к левому краю.

Теперь все тайлы одинаковой высоты, ряды ровные.

### 3. Объектный поиск

Новый таб **«Поиск»** ([`Features/Search/SearchView.swift`](../../LiboLibo/Features/Search/SearchView.swift), 4-й в `TabView` между Подкастами и Моё):

- Стандартный `.searchable` со строкой `.navigationBarDrawer(displayMode: .always)`.
- Один запрос — два списка результатов, сгруппированных по типу объекта:
  - **Подкасты** — фильтр по имени (`localizedLowercase.contains`).
  - **Выпуски** — фильтр по `title` или `summary` любого выпуска из `repository.allEpisodes`, топ-50.
- Тап по подкасту → `PodcastDetailView`. Тап по выпуску → запускается воспроизведение (та же кнопка, что и в Фиде).
- Пустые состояния: `ContentUnavailableView` с приглашением, когда запрос пуст; `ContentUnavailableView.search(text:)` когда ничего не нашлось.
- Авто-капитализация и автокоррекция отключены — для русских названий полезнее.

## Программная проверка тапа

`cliclick` всё ещё ругается «Accessibility privileges not enabled» — выданное разрешение не применилось к тому процессу, который запускает мои bash-команды. Конкретный пункт для включения нужно искать в **System Settings → Privacy & Security → Accessibility**: имя бинарника, из которого запускается Claude Code (Terminal / iTerm / приложение Claude Code). Имеет смысл проверить, не нужно ли разрешить **отдельно тот шелл, что я использую** (zsh в окне, где работает claude). Как заработает — буду тапать табы и кнопки сам и снимать скриншоты других экранов автоматически.

## Что Илья видит при тестировании

1. Лента, тап по выпуску — играет.
2. Mini-player «прилипает» к зоне над таб-баром, использует системный glass, не наезжает.
3. Скролл вниз — таб-бар сжимается автоматически, mini-player остаётся.
4. На «Подкастах» — ровные ряды, без сдвинутой левой колонки.
5. Таб «Поиск» — поле ввода, две секции «Подкасты» и «Выпуски» по мере набора.
